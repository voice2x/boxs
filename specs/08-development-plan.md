# 开发计划与成本

## 一、开发阶段

| 阶段 | 周期 | 交付内容 |
|------|------|---------|
| **P0 原型** | 3 周 | iOS MVP（Swift）— 语音输入 → NLU 意图识别 → 记账/打卡/备忘基础 CRUD |
| **P1 完善** | 2 周 | Android 移植（Kotlin）+ 统计图表 + 拍照 OCR + 提醒通知 + UI 美化 |

| **P3 迭代** | 持续 | AI 对话查询、微信小程序、Widget、数据云同步 |

总计 **6 周可上线 iOS MVP**。

## 三、成本估算

### API 成本

| 项目 | 费用 |
|------|------|
| LLM API（GPT-4o-mini） | ~$0.15/百万 input token，每天 100 条 ≈ $0.01/用户/月 |
| LLM API（GLM-4-Flash） | 更便宜，部分免费额度 |
| STT（讯飞） | 免费额度 500 次/天，超出 ¥0.0034/次 |
| OCR（百度） | 通用文字 1000 次/天免费 |

### 单用户月成本

由于规则引擎处理约 60-70% 的请求，实际 LLM 调用量大幅降低：

| 场景 | 比例 | 处理方式 | 单次成本 | 延迟 |
|------|------|---------|---------|------|
| 简单记账 "午饭35" | 40% | 规则引擎 | ¥0 | <10ms |
| 简单打卡 "跑步打卡" | 20% | 规则引擎 | ¥0 | <10ms |
| 复杂记账 "打车花了28，后来又买了杯咖啡" | 20% | LLM (glm-4-flash) | ~¥0.001 | ~200ms |
| 查询 "这个月餐饮花了多少" | 15% | LLM (gpt-4o-mini) | ~¥0.002 | ~500ms |
| 无法识别 | 5% | 提示重新输入 | ¥0 | 0ms |

**单个活跃用户月成本：< ¥0.5**

### 服务器成本

初期可无（纯本地 + 免费 API 额度）。后续按需：

| 规模 | 方案 | 费用 |
|------|------|------|
| < 1000 用户 | 无服务器（纯本地 App） | ¥0 |
| 1000-10000 用户 | 轻量云函数（云同步） | ~¥100/月 |
| 10000+ 用户 | 云服务器 + 数据库 | ~¥500/月 |

## 四、变现模式

| 模式 | 价格 | 说明 |
|------|------|------|
| 免费版 | 免费 | 基础记账+打卡+备忘，每日 10 次 AI 语音 |
| Pro 订阅 | ¥12/月 或 ¥98/年 | 无限语音、AI 对话查询、OCR 票据、云同步、高级图表 |
| 一次性买断 | ¥68 | 终身基础功能（不含 AI 高级功能） |

### 免费版限制设计

每日 10 次 AI 语音的限制是核心转化点：

- 规则引擎处理的请求**不计数**（"午饭35" 这类不算）
- 只有走 LLM 的请求才消耗次数
- 重度用户很快会用完，自然转化
- 轻度用户（每天 < 10 条复杂输入）可以免费用
